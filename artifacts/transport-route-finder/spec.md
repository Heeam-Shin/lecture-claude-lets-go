# spec — 서울 교통 최적 경로 탐색기

## 개요

서울시 실시간 지하철 도착정보와 따릉이 대여정보를 바탕으로, 출발지에서 목적지까지 도보·지하철·따릉이를 조합한 최단 시간 경로 상위 3개를 도식적으로 보여준다.

## 범위

### 포함

- 출발지·목적지 텍스트 검색 입력
- 지우기 버튼으로 입력·결과 전체 초기화
- 최단 시간 기준 상위 3개 경로 결과 표시
- 각 경로: 도식적 다이어그램 + 세부 구간별 교통수단 + 세부 구간별 이동시간 + 전체 이동시간
- 도보 구간 포함 (대여소·역까지 걷는 시간 반영)
- 따릉이 대여 불가(잔여 0대) 경우 경로는 표시하되 대여 불가 상태 표시
- 실시간 API 오류 시 에러 메시지 표시

### 제외

- 버스·택시 등 다른 교통수단 — 서울시 공개 API 범위 밖 (v1 범위 외)
- 지도 클릭으로 출발지·목적지 선택 — 텍스트 검색만으로 v1 기능 충족 가능
- 로그인·즐겨찾기·검색 기록 — 단순 조회 서비스로 계정 관리 불필요
- 과금·결제 흐름 — 무료 서비스

## 시나리오

### 1. 경로 검색 — 정상

- **Given** — 출발지와 목적지가 모두 입력된 상태
- **When** — 검색 버튼을 클릭한다
- **Then** — 전체 이동시간이 짧은 순서대로 경로 3개가 표시되고, 각 경로에는 세부 구간별 교통수단(도보/지하철/따릉이)과 이동시간이 나타난다

성공 기준:
- [ ] 출발지 "서울역", 목적지 "강남역" 입력 후 검색 → 경로 카드 최대 3개가 렌더된다
- [ ] 각 경로 카드에 전체 이동시간(분)이 표시된다
- [ ] 각 경로 카드에 세부 구간(도보/지하철 노선명 또는 따릉이)과 구간별 이동시간이 표시된다
- [ ] 경로 카드는 전체 이동시간 오름차순으로 정렬된다

### 2. 경로 다이어그램

- **Given** — 경로 검색 결과가 표시된 상태
- **When** — 경로 카드를 본다
- **Then** — 각 경로는 도식적 다이어그램으로 구간 흐름(출발 → 구간1 → 구간2 → … → 도착)을 시각화한다

성공 기준:
- [ ] 각 구간에 교통수단 아이콘(도보·지하철·따릉이)이 표시된다
- [ ] 지하철 구간에는 노선명(예: 1호선)이 표시된다
- [ ] 따릉이 구간에는 대여소 이름이 표시된다
- [ ] 다이어그램이 출발지에서 목적지까지 좌→우 또는 위→아래 순서로 읽힌다

### 3. 지우기 버튼

- **Given** — 출발지·목적지가 입력되어 있고 검색 결과가 표시된 상태
- **When** — 지우기 버튼을 클릭한다
- **Then** — 출발지 입력 필드, 목적지 입력 필드, 검색 결과가 모두 비워진다

성공 기준:
- [ ] 지우기 클릭 후 출발지 필드가 비어 있다
- [ ] 지우기 클릭 후 목적지 필드가 비어 있다
- [ ] 지우기 클릭 후 경로 결과 카드가 화면에 없다

### 4. 입력 불완전 — 검색 불가

- **Given** — 출발지 또는 목적지 중 하나가 비어 있는 상태
- **When** — 검색 버튼을 클릭한다
- **Then** — 검색이 실행되지 않고, 비어 있는 필드에 입력이 필요하다는 안내가 표시된다

성공 기준:
- [ ] 목적지 비어 있을 때 검색 버튼 클릭 → 경로 결과가 나타나지 않는다
- [ ] 목적지 비어 있을 때 검색 버튼 클릭 → 입력 필요 안내 문구가 표시된다

### 5. 실시간 API 오류

- **Given** — 출발지·목적지가 입력된 상태
- **When** — 검색 버튼을 클릭했으나 실시간 API(지하철 또는 따릉이)가 응답하지 않는다
- **Then** — "실시간 데이터를 가져오지 못했습니다" 에러 메시지가 표시되고, 이전에 표시되던 경로 결과는 사라진다

성공 기준:
- [ ] API 오류 시 에러 메시지가 화면에 표시된다
- [ ] API 오류 시 이전 경로 결과가 화면에 남아 있지 않다

### 6. 경로 없음

- **Given** — 출발지·목적지가 입력된 상태
- **When** — 검색 버튼을 클릭했으나 지하철·따릉이 어떤 조합으로도 경로를 찾을 수 없다
- **Then** — "경로를 찾을 수 없습니다" 메시지가 표시된다

성공 기준:
- [ ] 서울 서비스 범위 밖 지역 입력 → 경로 없음 메시지가 표시된다
- [ ] 경로 없음 메시지 표시 시 경로 카드는 나타나지 않는다

### 7. 따릉이 대여 불가

- **Given** — 경로 중 따릉이 구간의 출발 대여소 잔여 자전거가 0대인 상태
- **When** — 해당 경로가 결과에 포함되어 표시된다
- **Then** — 경로 카드는 표시되지만 해당 따릉이 대여소에 "대여 불가" 표시가 나타난다

성공 기준:
- [ ] 따릉이 잔여 0대 대여소가 포함된 경로 → 경로 카드가 표시된다
- [ ] 해당 경로 카드의 따릉이 대여소 영역에 "대여 불가" 상태가 표시된다

## 불변 규칙

**실시간 데이터 신선도**: 화면에 표시되는 지하철 도착 정보와 따릉이 잔여 대수는 검색 시점의 실시간 API 응답값을 기반으로 한다. 캐시된 이전 결과를 보여주지 않는다.

## 의존성

- 서울시 지하철 실시간 도착정보 API (https://data.seoul.go.kr/dataList/OA-12764/A/1/datasetView.do)
- 출발지·목적지 텍스트 → 좌표 변환이 가능한 지오코딩 서비스
